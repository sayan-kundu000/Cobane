# Security Policy — Cobane

We take security vulnerabilities seriously. Please do not report security issues via public GitHub issues. Instead, follow the private disclosure process described below.

---

## 1. Supported Versions

Security updates are actively applied to the following project versions:

| Version | Supported          |
| ------- | ------------------ |
| v1.0.x  | :white_check_mark: |
| < v1.0  | :x:                |

---

## 2. Reporting a Vulnerability

If you discover a security vulnerability, please send a detailed private email to:

**kundusayan2002@gmail.com**

Please include in your report:
- A description of the vulnerability, including severity rating.
- Step-by-step instructions to reproduce the issue (including any payload triggers or API headers).
- Any potential impact on users, databases, or AI API key keys leakage.

We will acknowledge receipt of your report within 48 hours and work with you to analyze, resolve, and publish a patch before disclosure.
